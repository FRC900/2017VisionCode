
Example training run.

POS count : consumed  is the number of positive samples used. This will start at numPos for the first stage of training and gradually increase for later stages.

NEG is the number of negative samples used. This will be numNeg.  
Negatives are only used if they are not rejected by previous stages.  acceptanceRatio is the number of negative images used for this stage (i.e. not rejected by previous stages) as a percentage of the total number searched. Since the false positive rate is 0.5 for our training, this should start at 1 for the first stage and decrease by a factor of 2 (roughly) for each subsequent stage.

The output columns are as follows :

N = iteration count. This goes up from 1 to maxWeakCount. It will also exitif other limits are satisfied - hit rate > 0.999 and false alarms <= 0.50.
HR = hit rate. This is the ratio of positive images successfully detected by the current iteration of this stage of training.
FA = false alarm rate = % of false positives (negative samples) accepted. This will gradually decrease
